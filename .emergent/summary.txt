<analysis>**original_problem_statement:**
The user runs a Shopify store (www.allbobbleheads.com) selling custom bobbleheads. They need a system for customers to approve proofs at two stages: clay and paint.

**PRODUCT REQUIREMENTS:**
1.  **Customer Portal:** Customers should be able to log in (using email and order number) to a dashboard to see their orders.
2.  **Order Details View:**
    *   View the order's current stage (e.g., Clay, Paint, Shipped).
    *   View proof images for the current stage, with the ability to enlarge them.
    *   Buttons to Approve or Request Changes.
3.  **Approval/Changes Workflow:**
    *   **Approve:** Clicking Approve moves the order to the next stage and sends an email notification to the admin.
    *   **Request Changes:** Clicking Request Changes opens a text box for comments and allows image attachments. This action also sends an email notification to the admin.
4.  **Admin Dashboard:**
    *   An internal dashboard for the store owner to manage orders.
    *   Ability to manually upload proof images (or a zip file) for each order.
    *   View order statuses and customer feedback.
    *   Initially, a way to sync orders from Shopify.
5.  **Logging:** All status changes and customer inputs must be tracked in a Google Sheet.
6.  **Security:** Customers should only be able to see their own orders.
7.  **UI/UX:** The user requested specific UI layouts, including a drag-and-drop uploader for admins and branding elements from their main website.

**what currently exists?**
A full-stack application with a React frontend and FastAPI backend has been built.
-   **Frontend:** Features a customer portal for order lookup (email + order #) and an admin dashboard. The customer portal allows viewing proofs, approving, or requesting changes. The admin dashboard allows syncing Shopify orders, viewing all orders, searching/filtering, uploading proofs via drag-and-drop, manually changing order stage/status, and pinging customers. The UI has been customized with the user's logo and branding.
-   **Backend:** Handles API requests for all frontend functionality.
-   **Integrations:**
    -   **Shopify:** Syncs orders and customer data.
    -   **Google Sheets:** Logs all major events (proof uploads, approvals, status changes).
    -   **SMTP:** Sends email notifications to the admin for customer actions and allows the admin to send ping reminders to customers.
-   **Database:** MongoDB stores order data, including stages, statuses, proof images (as base64), and an interaction timeline.
